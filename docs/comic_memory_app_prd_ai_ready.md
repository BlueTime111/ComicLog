# 漫画剧情记忆与分享 App｜PRD（面向 AI 开发）

## 0. 文档使用说明
本 PRD 以“可交付、可拆解、可验收”为原则编写，目标不是展示创意，而是作为后续 AI 辅助开发的单一需求来源。文档中的模块边界、实体定义、接口草案、交互规则和验收标准，均应被视为优先参考。

### AI 开发约束
- 若 AI 在执行过程中发现需求缺失，应优先按本文约定的默认规则处理，并在输出中显式列出“假设项”。
- 不要因为局部模糊而重写产品方向。
- 先满足 P0/P1 范围，再考虑 P2/P3。

## 1. 产品概述
- **产品名称**：漫画剧情记忆与分享 App
- **产品一句话**：帮助漫画读者记录、回顾并交换某部漫画的章节级总结。
- **MVP 核心**：章节总结、已读进度、读前回顾、总结包导出、总结包导入、共享层/私人层分离。
- **核心原则**：local-first、低录入成本、防剧透、结构化分享、可持续维护。

## 2. 目标、非目标与优先级
### 2.1 产品目标
- 降低追更后的回忆成本，让用户在继续阅读前快速找回剧情上下文。
- 通过总结包导入/导出降低个人维护成本，形成共享数据层。
- 先完成本地可用的 MVP，再逐步扩展账号、链接分享和订阅更新。

### 2.2 非目标
- 不做漫画阅读器
- 不做资源站
- 不做公共全文 OCR 库

### 2.3 需求优先级
- **P0**：作品库、章节总结、已读进度、回顾、导入/导出、分层存储
- **P1**：搜索、导入预览、冲突处理、分享包元信息展示
- **P2**：账号体系、链接分享、包版本订阅更新
- **P3**：OCR、AI 摘要、角色关系图、社区广场

## 3. 用户角色与术语
- **作品（Series）**：一部漫画，如《海贼王》或《咒术回战》。
- **章节（Chapter）**：作品中的单话/章节/番外，编号使用字符串存储。
- **章节总结（Chapter Summary）**：用户为某一话维护的一句话或短摘要，以及标签、角色等结构化信息。
- **总结包（Summary Pack）**：围绕某部作品整理的一组章节总结，可导出、分享、导入。
- **共享层（Shared Layer）**：来自总结包的公共内容，可被更新与同步。
- **私人层（Private Layer）**：当前用户自己的补充说明、标签、纠正与备注。
- **已读进度（Reading Progress）**：用户读到哪一话，用于防剧透与回顾范围控制。

## 4. 信息架构
- 首页 / 我的作品：展示用户维护或导入过的作品。
- 作品详情页：作品信息、进度、章节列表、最近几话回顾入口、导出入口。
- 章节详情页：共享层总结、私人备注、标签、角色和编辑入口。
- 导入页：读取文件或链接，预览包信息、覆盖范围、冲突情况，再执行导入。
- 搜索页：按作品内关键词、角色、事件或标签搜索已导入/已维护内容。

## 5. MVP 功能范围
- **FR-01** 作品库管理
- **FR-02** 章节管理与总结
- **FR-03** 读前回顾
- **FR-04** 总结包导出
- **FR-05** 总结包导入
- **FR-06** 共享层 / 私人层
- **FR-07** 搜索

## 6. 用户流程
### 6.1 首次使用流程
1. 进入首页，看到空状态：可新建作品，也可导入总结包。
2. 若已有别人维护的包，优先导入；若没有，则手动新建作品。
3. 为作品设置已读到哪一话。
4. 系统展示最近几话回顾；用户也可继续为新章节补充总结。

### 6.2 分享流程
1. 用户在作品详情页点击“导出总结包”。
2. 系统生成结构化文件，并写入包元信息、覆盖范围和版本号。
3. 用户通过系统分享或文件发送给其他人。
4. 接收方在导入页读取文件并预览差异。
5. 确认后导入共享层，本人可继续写私人备注。

### 6.3 导入冲突流程
1. 系统比较导入包与本地当前共享层，计算新增、更新、冲突章节数。
2. 用户选择导入策略：仅导入缺失章节 / 覆盖共享层 / 逐条查看冲突。
3. 执行导入：共享层被更新；私人层保持不变。
4. 导入完成：显示导入结果，并允许用户立即查看更新了哪些话。

## 7. 功能需求明细
### 7.1 FR-01 作品库管理
**输入字段**
- 作品标题（必填）
- 别名（可选）
- 封面（可选）
- 语言（可选）
- 备注（可选）

**核心动作**
- 创建作品
- 编辑作品
- 删除/归档作品
- 设置当前已读进度

**验收标准**
- 填写作品标题后可成功创建记录
- 用户可将已读进度调整到任意存在的章节号
- 无作品时必须明确引导“新建作品”与“导入总结包”

### 7.2 FR-02 章节管理与总结
**最小字段集**
- 章节号（必填，字符串）
- 章节标题（可选）
- 一句话总结（建议）
- 关键词标签（可选）
- 主要角色（可选）
- 已读状态

**规则**
- 默认展示一句话总结输入框，其他字段可折叠
- 不得强制将章节号解析为整数
- 章节详情页优先展示合并后的摘要：共享层在前，私人层补充在后

**验收标准**
- 任意作品下可新增章节并保存
- 可保存“12.5”“番外”“特别篇”等非整数编号
- 保存后可再次进入并修改，不丢失历史内容

### 7.3 FR-03 读前回顾
**规则**
- 至少支持上一话摘要、最近 3 话摘要聚合、当前篇章摘要占位
- 绝不能展示超过用户已读进度的章节内容
- 用户进入作品详情页时可看到回顾卡；也可在继续阅读前主动点击
- 优先取共享层摘要；若有私人备注，则以“补充说明”的方式附加

**验收标准**
- 用户已读到第 30 话时，回顾中不得出现第 31 话及之后的内容
- 系统能从最近 N 话摘要中拼接出简要回顾内容
- 若尚无摘要，提示用户补充，而不是展示空白页面

### 7.4 FR-04 总结包导出
**规则**
- 以某部作品为单位导出共享层数据，默认不导出用户私人层
- MVP 推荐 JSON，便于调试和 AI/开发工具读取
- 元信息必须包含 schemaVersion、packId、title、author、version、updatedAt、coverage
- 作品详情页中提供显式导出按钮

**验收标准**
- 导出文件包含作品信息、包信息、章节数组
- 导出的文件可以被同一 App 的导入流程再次识别
- 若无额外配置，默认不导出私人备注

### 7.5 FR-05 总结包导入
**规则**
- MVP 先支持从本地文件导入；链接导入放在后续版本
- 导入前必须显示作品名、作者、覆盖范围、版本号、章节数、变更摘要
- 支持仅导入缺失章节、覆盖共享层、逐条查看冲突
- 导入结果应提示新增/更新记录数，以及是否保留本地私人层

**验收标准**
- 无 schemaVersion 或关键字段缺失时，拒绝导入并提示错误
- 正式写入前，用户必须能查看元信息与变化范围
- 任意导入策略下，私人层都不应被覆盖

### 7.6 FR-06 共享层 / 私人层
**规则**
- 共享层与私人层必须独立存储
- 章节页显示时进行合并：共享层为基础信息，私人层显示为“我的补充”或“我的备注”
- 未来共享包版本更新时，仅替换共享层；私人层保持不变
- 用户可删除私人层内容，不应影响共享层数据

**验收标准**
- 数据库或本地存储中能明确区分共享记录与私人记录
- 同一章节既有共享摘要又有私人备注时，可同时展示
- 重新导入新版本包时，私人备注完整保留

### 7.7 FR-07 搜索
**规则**
- 检索范围：当前用户本地可见的作品、章节摘要、角色、标签
- 搜索字段：作品名、章节标题、摘要内容、标签、角色
- 结果中返回作品、章节号、命中字段摘要，点击后跳转到章节详情
- MVP 只要求本地 SQLite 文本检索

**验收标准**
- 输入关键词后可返回相关章节
- 结果中能看到关键词命中在哪里
- 无网络时搜索功能仍可使用

## 9. API 草案（后续云端版本）
MVP 可以完全本地运行。本节作为后续多人分享或接入后端时的接口草案。

- `GET /series`：获取作品列表
- `POST /series`：创建作品
- `GET /series/{id}/chapters`：获取章节列表
- `POST /series/{id}/chapters`：创建章节
- `POST /series/{id}/export`：生成总结包文件或 JSON 内容
- `POST /packs/import`：上传并解析总结包，返回预览与冲突摘要
- `POST /packs/{packId}/apply`：按选定策略正式导入
- `GET /search`：按关键词检索章节摘要

## 10. 默认交互与界面要求
- 无作品时，主按钮应为“导入总结包”；次按钮为“新建作品”
- 章节录入先展示章节号与一句话总结；标签、角色等为二级字段
- 回顾卡要在作品详情页上方优先展示
- 导入前必须给出新增/更新/冲突统计
- 章节页中要把“我的补充”与共享内容视觉区分开

## 11. 非功能需求
- 离线能力：MVP 主要流程需可离线运行，包括阅读、编辑、搜索、导入本地文件
- 数据安全：导入失败不得破坏现有本地数据；重要写入需有事务或回滚策略
- 性能：作品列表、章节列表和本地搜索在常见数据量下应流畅可用
- 兼容性：优先支持 iOS / Android；Web 为可选后续能力
- 可迁移性：数据模型需便于后续引入云同步，不把逻辑写死在 UI 层

## 12. 错误状态与边界情况
- 导入文件损坏：提示“文件不可识别或已损坏”，不写入任何数据
- schemaVersion 不支持：提示版本不兼容，并显示当前支持版本
- 章节号冲突但内容不同：进入冲突预览，要求用户选择策略
- 用户删除共享包：只删除共享层引用，私人层仍可保留并提示来源已失效
- 无任何章节摘要：作品详情页显示引导，提示从最近一话开始补充

## 13. 测试与验收建议
- 单元测试：数据模型校验、导入解析、冲突合并、搜索索引生成
- 集成测试：创建作品 → 新增章节 → 导出 → 导入 → 查看回顾 → 搜索
- 边界测试：特殊章节号、空摘要、重复导入同一包、新旧版本切换
- 人工验收：重点验证低录入成本、防剧透、共享层/私人层不串数据

## 14. 推荐技术栈与实现建议
- 前端：React Native + Expo + TypeScript
- 状态管理：优先轻量状态方案；将 UI 状态与仓储层解耦
- 本地数据库：SQLite（local-first），搜索先做全文 / 关键词检索
- 后端（后续）：Supabase
- AI 能力（后续）：OCR 与摘要只做增强层，不进入 MVP 核心路径
